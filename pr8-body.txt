# PR8: 编辑器增强与状态持久化

## 本次变更

### 1. 本地状态持久化 (`lib/storage.ts`)

新增本地存储工具库，提供以下功能：

- `saveDraft()` - 保存剧本草稿到 localStorage
- `getDraft()` - 获取保存的草稿
- `deleteDraft()` - 删除草稿
- `saveFileDraft()` / `getFileDraft()` - 文件内容草稿
- `addRecentCharacter()` / `getRecentCharacters()` - 最近使用的角色列表

### 2. 智能 Mock 数据生成 (`lib/smartMock.ts`)

全新的智能 mock 数据生成器，能够：

- **智能标题提取** - 自动识别章节标题
- **角色自动识别** - 从对话中提取角色名
- **时间关键词提取** - 识别白天、夜晚、清晨等时间
- **地点关键词提取** - 识别房间、街道、餐厅等地点
- **对话智能解析** - 从引号中提取对话内容
- **情感音调推测** - 根据语气词自动判断情感

### 3. 首页草稿恢复 (`app/page.tsx`)

- 页面加载时自动检测是否有未保存的草稿
- 弹窗询问用户是否恢复草稿
- 编辑过程中自动保存草稿（每5秒或每次更改）
- 成功保存到数据库后自动清除本地草稿
- 使用 Toast 提示草稿恢复状态

### 4. Mock 生成器集成 (`lib/modelscope.ts`)

- 所有 fallback 场景都使用智能 mock 生成器
- AI 调用失败时生成更真实的示例数据
- 包含详细的日志输出便于调试

## 核心改进

- **刷新不丢失** - 编辑内容自动保存到本地，刷新后可恢复
- **更真实的示例数据** - mock 数据基于实际小说内容生成
- **智能内容识别** - 自动提取角色、时间、地点等信息
- **无感知保存** - 后台自动保存，无需手动操作

## 技术实现

### 存储策略
```
用户操作 → 自动保存到 localStorage → 保存到数据库 → 清除 localStorage
```

### 智能提取流程
```
小说文本 → 标题/角色/时间/地点 → 场景划分 → 对话/动作提取 → 情感推测
```

## 验证方式

1. 刷新页面，测试草稿恢复功能
2. 编辑剧本后刷新页面，检查是否提示恢复
3. 测试保存到数据库后草稿是否清除
4. 在 AI 调用失败时查看生成的 mock 数据是否基于小说内容
