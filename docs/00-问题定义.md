# 问题定义

## 最终交付物
完整 AI 剧本创作工具（Web 应用）：
1. 用户系统（注册/登录/API Key 管理）
2. 文件上传（txt/docx）
3. AI 转换（ModelScope DeepSeek-V4-Flash）
4. 可视化编辑器（编辑场景、对话、动作、旁白、情感音调）
5. YAML 预览/下载
6. 历史记录（查看、恢复）
7. 角色管理（集中管理）

## 硬约束
- 必须使用 ModelScope DeepSeek-V4-Flash
- 必须安全存储 API Key（后端代理）
- 必须本地可运行
- 必须按 7 个 PR 分批提交

## 隐含要求
- YAML Schema 完整
- 剧本要素齐全：时间、场景、人物、对话、动作、旁白、情感音调
- 支持 3 章节以上小说
- 界面简洁高效

## 什么叫完成
用户能完成完整流程：
1. 注册/登录
2. 上传小说
3. AI 生成剧本
4. 可视化编辑
5. 保存历史
6. 下载 YAML

## 最容易翻车的风险点
1. AI 转换质量不稳定
2. API 调用成本控制
3. 大文件处理性能
4. 前后端通信安全
5. 本地存储容量限制
