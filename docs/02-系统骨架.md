# 系统骨架设计

## 模块拆分
```
1. 项目初始化模块 - Next.js 项目搭建 + 配置
2. YAML Schema 模块 - 剧本结构定义 + 文档
3. 文件上传模块 - 前端上传 + 后端解析
4. AI 转换模块 - ModelScope API 调用 + 提示词工程
5. 可视化编辑模块 - React 组件 + YAML 双向绑定
6. 用户系统模块 - NextAuth.js + API Key 管理
7. 历史记录模块 - Prisma SQLite + CRUD
8. 角色管理模块 - 角色数据管理
```

## 各模块职责
| 模块 | 职责 |
|------|------|
| 项目初始化 | Next.js + Tailwind + Prisma + NextAuth 配置 |
| YAML Schema | 剧本结构定义、设计文档 |
| 文件上传 | 文件选择、内容读取（txt/docx） |
| AI 转换 | 调用 ModelScope API、提示词设计、YAML 解析 |
| 可视化编辑 | React 表单编辑、实时 YAML 同步 |
| 用户系统 | 登录/注册、API Key 加密存储 |
| 历史记录 | 剧本的保存、列表、恢复、删除 |
| 角色管理 | 角色的增删改查 |

## 核心接口
| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/convert` | POST | 上传小说文本，返回 YAML 剧本 |
| `/api/scripts` | GET | 获取剧本列表 |
| `/api/scripts` | POST | 保存新剧本 |
| `/api/scripts/[id]` | GET | 获取单个剧本 |
| `/api/scripts/[id]` | PUT | 更新剧本 |
| `/api/scripts/[id]` | DELETE | 删除剧本 |

## 关键数据链路
```
用户上传文件
→ 前端读取文件内容
→ POST /api/convert
→ 后端调用 ModelScope
→ 返回 YAML
→ 解析为 JSON
→ 前端渲染编辑器
→ 用户编辑
→ PUT /api/scripts/[id]
→ Prisma 保存到 SQLite
```

## 主链路时序
```
1. 用户登录/注册
2. 进入上传页面，选择小说文件
3. 前端读取文件内容，POST /api/convert
4. 后端代理调用 ModelScope API（使用用户存储的 API Key）
5. AI 返回 YAML，后端验证格式
6. 前端渲染可视化编辑器，实时同步 YAML 预览
7. 用户编辑完成，点击保存，PUT /api/scripts/[id]
8. Prisma 保存到 SQLite
9. 用户可在历史记录页面查看、恢复、删除
```

## 权限/边界
- API Key 加密存储在用户表中，后端代理时使用
- 历史记录按用户隔离
- 未登录用户无法使用
